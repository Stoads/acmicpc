배열의 구간합 누적에 대한 문제이다.

간단한 아이디어로, 배열 A에 대해서 [3,5]에 값을 1 늘린다 치면, A의 변화량 배열 D에 대해서 D[3]+=1,D[6]-=1 하면 되는 개념이다.

값의 저장을 D로 하고, 최종적으로 D를 더해나가면서 A를 구하면 되는 방법이다.

막말로 이번엔 쿼리 누적 이후에 값을 물어봐서 상관이 없지만, 쿼리 중간중간 값을 구해야 할 경우에는 Segment Tree를 써야한다.


#include <cstdio>
#include <algorithm>
int N,K;
int arr[1234567];
int main(){
    scanf("%d%d",&N,&K);
    for(int i=0,A,B;i<K;i++){
        scanf("%d%d",&A,&B);
        arr[A-1]++;
        arr[B]--;
    }
    for(int i=1;i<N;i++)
        arr[i]+=arr[i-1];
    std::nth_element(arr,arr+N/2,arr+N);
    printf("%d",arr[N/2]);
}

처음 값의 차분은 없으니 0에서 시작 후, 입력받는 구간마다 차분을 구하고, 그걸 누적해서 원본 배열을 구하는 방식으로 하면 빠르게 구할 수 있다.
